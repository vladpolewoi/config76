### Раздел C — Риски, зависимости, рекомендации

Вывод: требования в целом описывают основное поведение отключения папки «Мои документы», но перед backend-разработкой БФТ нужно точечно привести к новой permission-модели. Сейчас описание выглядит как запрет по роли, а целевая модель должна быть разрешающей: доступ есть при наличии permission на «Мои документы».

1. Переписать БФТ с deny-логики «роль в списке отключения» на разрешающую permission-модель.

2. Зафиксировать permission на доступ к папке «Мои документы».

3. Описать возможность назначения нескольких ролей одной группе.

4. Зафиксировать, что permission «Мои документы» управляется через разрешающую permission-модель и может настраиваться для служебных ролей.

5. Зафиксировать, что Admin и SuperAdmin сохраняют доступ по умолчанию.

6. Не разблокировать полное редактирование служебных ролей; разрешить только изменение permissions для служебных ролей всех типов: «Все», Admin и SuperAdmin.

7. Привести ожидаемый код ошибки при отсутствии доступа к текущему backend-контракту: 406 NotAcceptable.

8. После этих правок повторно оценить трудозатраты и передавать задачу в backend-разработку.
